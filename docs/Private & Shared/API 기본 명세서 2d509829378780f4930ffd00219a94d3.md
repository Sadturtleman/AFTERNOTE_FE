- 파트 나누기
    
    ### 1. 👑 Member A (리드/본인): "코어 & 보안 담당"
    
    프로젝트의 뼈대를 잡고, 팀원들이 개발할 수 있는 환경을 만들어줍니다. 가장 어렵고 중요한 부분입니다.
    
    - **담당 도메인:**
        - **`Auth` (인증):** 카카오 로그인, JWT 발급/재발급, 로그아웃.
        - **`Afternote` (유산/유언장):** 이중 암호화(AES-256)가 필요한 민감 정보 처리.
        - **`Global / Infra`:** AWS 세팅(EC2, RDS, S3), CI/CD, 공통 예외 처리(`GlobalExceptionHandler`), BaseEntity 등.
    - **이유:** 보안과 인프라는 초반에 안 잡으면 나중에 다 뜯어고쳐야 합니다. 리드가 이걸 잡고 가는 게 안전합니다.
    
    ### 2. 📹 조영탁: "타임레터 & 미디어 담당"
    
    프로젝트의 메인 기능인 '편지'와 '파일'을 담당합니다.
    
    - **담당 도메인:**
        - **`Time-Letters` (타임레터):** 편지 작성, 조회, 공개일 로직.
        - **`Received` (수신자):** 로그인 안 한 유족이 `UUID`로 조회하는 로직.
        - **`S3 Utils`:** 프론트엔드에게 `Presigned URL`을 발급해 주는 공통 로직 구현.
    - **이유:** 영상 업로드 흐름과 비회원 접근(UUID) 로직은 비즈니스 로직 중 가장 복잡합니다. 한 명이 집중해서 개발해야 꼬이지 않습니다.
    
    ### 3. 📝 김소희: "마음의 기록 & 회원 담당"
    
    데이터 관리가 많고 화면이 아기자기하게 많은 부분을 담당합니다.
    
    - **담당 도메인:**
        - **`Mind-Record` (마음의 기록):** 데일리 질문, 일기 작성, 캘린더 월별 조회.
        - **`Users` (회원):** 마이페이지, 프로필 수정, 회원 탈퇴.
        - **`Report` (통계):** (여유가 된다면) 주간 리포트, 감정 통계 API.
    - **이유:** '일기'와 '질문'은 CRUD(생성/조회/수정/삭제)가 많습니다. 꼼꼼하게 API를 여러 개 찍어내야 하는 역할이라 성실한 분에게 맡기면 좋습니다.
