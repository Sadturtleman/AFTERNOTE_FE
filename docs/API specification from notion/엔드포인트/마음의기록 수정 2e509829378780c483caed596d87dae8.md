# 마음의기록 수정

카테고리: Mind-Record
설명: 기록 ID를 기준으로 내용을 수정합니다.
Method: PATCH
URL: /mind-records/{recordId}
header 포함 여부: 포함 O
백엔드 관리자: 김소희
상태: 개발 완료

기존에 작성된 마음의 기록을 수정하는 API입니다.

기록의 타입은 변경할 수 없으며, 내용만 수정할 수 있습니다.

### Request

**Path parameter**

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| recordId | 수정할 기록 ID | Long |  | X | 12 |

**Body parameter**

| **key** | **설명** | **value 타입** | **옵션** | **Nullable** | **예시** |
| --- | --- | --- | --- | --- | --- |
| title | 기록 제목 | String |  | O | 수정된 제목 |
| content | 기록 내용 | String |  | X | 수정된 내용 |
| date | 기록 날짜 | String | yyyy-MM-dd | X | 2026-01-12 |
| isDraft | 임시저장 여부 | Boolean |  | X | false |
| questionId | 데일리 질문 ID | Long | DAILY_QUESTION일 때만 | O | 3 |
| category | 깊은 생각 카테고리 | String | DEEP_THOUGHT일 때만 | O | 새로운 관점 |
- 기록 유형(type)은 수정할 수 없다.
- type = DIARY → questionId, category는 null
- type = DAILY_QUESTION → questionId만 포함
- type = DEEP_THOUGHT → category만 포함
- 임시저장(isDraft = true)인 경우에도 content는 빈 문자열("")로 전달될 수 있음

### Response

**Body**

| key | 설명 | value 타입 | 옵션 | Nullable | 예시 |
| --- | --- | --- | --- | --- | --- |
| recordId | 수정된 기록 ID | Long |  | X | 12 |

**Example**

```jsx
{
  "title": "수정된 제목",
  "content": "내용을 수정했습니다.",
  "date": "2026-01-12",
  "isDraft": false
}
```

### Status

| status | response content |
| --- | --- |
| 200 | 기록 수정 성공 |
| 400 | 잘못된 요청 또는 인증 오류 |
| 404 | 존재하지 않는 기록 |
